# Sales Data Analysis & MIS Dashboard

## Project Overview
This project analyzes retail sales data to identify revenue trends, product performance, regional growth, and inventory risks.

The objective was to simulate a real-world business environment by designing a relational database, performing SQL-based analysis, and building an MIS dashboard.

---

## Tools Used
- SQL Server
- Power BI
- Excel

---

## Database Design
The database consists of four tables:

1. Products
2. Stores
3. Sales
4. Inventory

Primary and Foreign key relationships were implemented to maintain data integrity.

---

## Key Analysis Performed
- Total Revenue & Profit Calculation
- Monthly Sales Trend
- Region-wise Performance
- Top 5 Products by Revenue
- Year-over-Year Growth
- Inventory Risk Identification

---

## SQL Techniques Used
- Joins
- Aggregations
- Subqueries
- CTE
- Window Functions (LAG, RANK)
- Views
- Indexing

---

## Dashboard Features
- KPI Cards (Revenue, Profit, Margin %)
- Monthly Trend Analysis
- Region Comparison
- Category Breakdown
- Top Products Analysis

---

## How to Run
1. Execute 1_database_setup.sql
2. Execute 2_sample_data_inserts.sql
3. Run analysis queries
4. Connect Power BI to SQL Server
